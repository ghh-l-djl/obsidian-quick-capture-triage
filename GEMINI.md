# GEMINI.md

## Project Context

This directory is an independent Obsidian community plugin, a companion to the rest of Obsidian Inbox. It does not call the `server/` API and does not touch git — it only reads and modifies files already present in the user's vault, specifically notes written by `server/` into the inbox folder.

The implementation was based on `../docs/superpowers/specs/2026-06-19-obsidian-inbox-plugin-design.md` and `../docs/superpowers/plans/2026-06-19-obsidian-inbox-plugin.md`. Treat the current source code as authoritative if those documents are stale.

## How To Work In This Directory

- Run plugin commands from this directory (`obsidian-plugin/`).
- Do not edit `node_modules/` or the build output `main.js` directly — it is generated by `npm run build`.
- `scanner.ts` and `actions.ts` must stay free of any runtime (non-`import type`) dependency on the `obsidian` package, so they remain testable with plain objects under `vitest` without the real Obsidian runtime.

Useful commands:

```bash
npm install
npm run build       # esbuild bundle -> main.js
npm run typecheck
npm test
```

## Plugin Architecture

- `src/main.ts`: `Plugin` entry — registers the view, ribbon icon, command, and settings tab.
- `src/view.ts`: `InboxTriageView` (`ItemView`) — sidebar list of untriaged inbox notes, with save/discard buttons.
- `src/scanner.ts`: pure scan logic — finds notes under `inboxFolder` with `status: inbox`, with `inboxTag` as a legacy fallback. No Obsidian runtime dependency.
- `src/actions.ts`: pure save/discard logic against minimal `FileManagerLike`/`VaultLike` interfaces. No Obsidian runtime dependency.
- `src/settings.ts`: `PluginSettings` type, `DEFAULT_SETTINGS`, and `InboxSettingTab`.

## Matching Rule

A note is "untriaged" when its path is under the configured `inboxFolder` and its frontmatter has `status: inbox`. For old notes, `inboxTag` remains a compatibility fallback checked against both frontmatter `tags` and inline `#tag`s in the note body. Saving a note changes `status` to `active` and strips the legacy inbox tag when present.

## Out of Scope

Do not add: calls back to the `server/` API, image/attachment handling on save or discard, per-note folder pickers, or a one-at-a-time modal review flow — see the design spec for the reasoning.
